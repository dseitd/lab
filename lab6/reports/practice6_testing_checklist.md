# ЛР6 — Чек‑лист тестирования (сайт: GitHub)

## Ключевые функции
- Вход в систему
- Регистрация
- Восстановление пароля
- Создание репозитория
- Создание issue
- Редактирование файла
- Поиск по платформе/репозиторию

## Чек‑лист и результаты

| ID | Функция | Шаги | Ожидаемый результат | Фактический результат | Статус |
|---|---|---|---|---|---|
| TC‑01 | Вход (валидные данные) | Перейти на github.com → Sign in → ввести корректные email/логин и пароль | Успешный вход, переход в профиль | Вход выполнен | Pass |
| TC‑02 | Вход (невалидные данные) | Ввести неверный пароль | Ошибка «Incorrect username or password» | Ошибка отображается | Pass |
| TC‑03 | Регистрация | Sign up → заполнить email/логин/пароль → подтвердить | Создан аккаунт, письмо подтверждения | Работает по инструкции | Pass |
| TC‑04 | Восстановление пароля | На Sign in → Forgot password → ввести email | Письмо восстановления, смена пароля | Письмо приходит | Pass |
| TC‑05 | Создание репозитория | Кнопка New → имя и настройки → Create | Репозиторий создан, доступен по URL | Создан успешно | Pass |

| TC‑08 | Поиск по платформе | Поле Search → запрос → фильтр Repositories | Найдены репозитории по запросу | Результаты корректны | Pass |
| TC‑09 | Поиск по репозиторию | Строка поиска в Code/Issues | Найдены совпадения в выбранном репо | Результаты корректны | Pass |

## Замечания
- UX последовательный; ошибка при входе информативна.
- Подтверждение действий (создание репо/issue, коммит) отображается в UI.
- Ограничения: часть сценариев требует подтверждённый аккаунт/права.
